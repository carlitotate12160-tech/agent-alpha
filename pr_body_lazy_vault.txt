## Summary

Two fixes in one PR: (1) lazy vault provider to fix import-time crash on Oracle, (2) scrub real client names from repo.

## Fix 1: Lazy SecretsVaultProvider

**Problem:** `secrets_vault_from_env()` was called eagerly at `main.py:44`. On Oracle where `AGENT_ALPHA_PG_DSN` is set, the selector took the Postgres branch and called `load_vault_key()` (fail-closed) AT IMPORT TIME → 7 collection errors.

**Fix:** Replace with `SecretsVaultProvider` (mirrors `StoreProvider`):
- `secrets_mgr = SecretsManager()` — module default, in-memory, no key needed
- `secrets_provider = SecretsVaultProvider()` — per-tenant, lazy
- Key loaded only on first `for_tenant()` call when a DSN-backed tenant runs
- `run_engagement_task` resolves `task_secrets` per-tenant via `secrets_provider.for_tenant(tenant_id)`
